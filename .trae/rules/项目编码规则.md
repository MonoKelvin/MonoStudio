## 首要规则

- **每次AI对话**：尽量不要还原之前已经手动修改的内容
- **AI删除文件、代码时**：删除文件的时候需要确认其他地方是否有引用，避免删除后导致应用崩溃。如果要删除代码片段，连同html、js、css等关联的代码都需要删除，除非其他地方有引用。

## 编码设计

- **代码风格**：使用 **2 个空格**进行缩进，不要使用 Tab 字符
- **组件职责单一和复用**：尽量复用软件提供的通用组件，如果新增的组件可能通用也需要提取出来作为通用组件使用，大的页面可以拆分多个组件分文件编码，业务和UI分离。
- **代码注释**：在代码中添加必要的注释，包括函数、类、组件等。
- **代码质量**：编码完成后要检查是否逻辑、编码问题，确保功能正确无误易读且容易维护。

## UI 设计规范

- **布局**：采用响应式布局，支持不同屏幕尺寸，合理设计对齐、间距、填充等。注意：内容应该精简，减少冲报复描述的文字、标题或者无意义的内容，比如大标题已经有文字描述了，就不需要再重复描述。
- **字体**：使用系统默认字体，最小的字体应该使用`-md`结尾的字体大小变量，除非有必要用更小的字体，避免使用自定义字体。
- **风格**：高级简约，现代化 UI 设计，统一的样式，使用代码中定义的 CSS 变量和主题变量。需要支持深浅模式。
- **动画**：流畅的动画效果，不要冗余无效动画。
- **图标**：优先复用已有的 SVG 格式图标，避免重复创建，新增的图标放到 `assets/icons` 目录下
- **提示**：提示使用软件通用组件 `ToastService` 或者 `BasePromptDialog` 组件；tooltip使用 `v-tooltip` 来显示。

## 性能优化

- **懒加载**：按需加载组件和模块。加载大量数据时，避免一次性加载所有数据，滚动到底部时再加载更多数据。
- **异步和多线程**：处理大量、复杂数据时，可以采用异步和多线程技术，如需要频繁刷新的数据离开页面时停止刷新，进入页面时异步加载数据，不卡住界面，使用正在加载等描述占位。
